# TableOfContents

A scroll-spy navigation sidebar that scans a content container for `h2`/`h3` headings and links to each one, bolding whichever section is currently in view.

## Props

| Prop          | Type      | Default                      | Description                                                                 |
|---------------|-----------|-------------------------------|-------------------------------------------------------------------------------|
| `containerId` | `string`  | `'docs-content'`              | The `id` of the element to scan for `h2`/`h3` headings.                       |
| `label`       | `string`  | `"What's on this page?"`      | Heading text rendered above the list of links.                                |
| `className`   | `string`  | `''`                          | Additional classes applied to the wrapping `<nav>` element.                   |

`TableOfContents` also extends `HTMLAttributes<HTMLElement>` (minus `children`), so any native `<nav>` attribute (e.g. `id`, `data-*`) can be passed through.

## Behavior

- On mount (and whenever the route changes, via `usePathname` from `next/navigation`), scans `#{containerId}` for every `h2` and `h3` element.
- Headings inside `[data-toc-exclude]` or `[role="tabpanel"]` are skipped. Use `data-toc-exclude` to opt a section out (for example, content that already has its own in-page navigation), and `role="tabpanel"` is skipped automatically since tabbed content isn't part of the page's linear reading order.
- Any matched heading without an existing `id` gets one auto-generated by slugifying its text content (lowercased, non-alphanumeric characters stripped, spaces collapsed to hyphens). Duplicate slugs are disambiguated with a numeric suffix.
- Matched headings get `scroll-mt-20` added so they don't sit flush against the top of the viewport when scrolled to.
- If the container has no matching headings, the component renders `null`.

## Interactive States

- **hover**: inactive links shift from `text-text-muted` to `text-text` via `motion-safe:transition-colors`.
- **focus-visible**: `focus-visible:ring-2 focus-visible:ring-offset-2 focus-visible:ring-brand-ring`; the default outline is suppressed only in favor of this ring.
- **active section**: the heading currently in view (tracked via scroll position, not `IntersectionObserver`, so a short trailing section near the bottom of the page still gets marked active) renders `font-semibold text-text` and `aria-current="location"`.
- **click**: clicking a link scrolls the corresponding heading into view (`smooth`, or `auto` when `prefers-reduced-motion: reduce` is set), updates the URL hash via `history.pushState` without a full navigation, and immediately marks that link active.

## Accessibility

- Wrapped in `<nav aria-label="Table of contents">` so screen readers announce it as a distinct navigation landmark, separate from the page's primary navigation.
- The active heading is communicated both visually (`font-semibold text-text`) and via `aria-current="location"`, not by color alone.
- Links are real `<a href="#...">` elements: keyboard-focusable, and usable even if JavaScript fails to intercept the click (the browser's native hash-jump still works).
- Focus state is visible via `focus-visible:ring-2`, keyboard navigation only.

## Tokens Used

| Token                  | Applied to                                          |
|------------------------|------------------------------------------------------|
| `text-text`            | Active link, current section                          |
| `text-text-muted`      | Inactive links at rest; hover target is `text-text`   |
| `text-text-subtle`     | The label above the list                              |
| `--radius`             | Link corner radius                                    |
| `--duration-fast`      | Hover color transition                                 |
| `brand-ring`           | Focus-visible ring color                               |

## When to Use

- Use on long-form documentation or article pages that have multiple `h2`/`h3` sections, so readers can jump directly to a section and see where they are as they scroll.
- Requires a content container with a stable `id` matching `containerId`; without one, the component renders nothing.
- Not a replacement for primary site navigation or breadcrumbs: it only reflects the structure of the current page.
- Best paired with a wide viewport (it's commonly hidden below `xl:` in a docs shell and shown as a fixed-width rail alongside the main content column); on narrow viewports, prefer an in-page "Jump to section" affordance instead.

## Installation

```
npx dafink-ui add table-of-contents
```
